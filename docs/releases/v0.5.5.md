DazPilot is an AI-assisted desktop companion for DAZ Studio: chat-driven scene work, multi-agent task routing, asset tools, and a TCP bridge plugin that runs inside Daz.

This release focuses on **agent tooling in the UI**, a **cleaner app shell**, **reliable Windows bridge builds**, and **release pipeline fixes** so installers publish correctly on GitHub.

## Highlights

- **Agents panel** — browse the agent tree, inspect capabilities, test routing, and register custom sub-agents from the app.
- **Daz ↔ app handoff** — send viewport context from the bridge pane to the desktop app (app inbox on `127.0.0.1:8766`).
- **Bridge command manifest** — C++ plugin commands stay aligned with Rust MCP schemas (`bridge_commands.manifest` + CI parity test).
- **Scene & render UX** — scene presets in the Scene panel; render queue panel in the viewport.
- **Multi-platform installers** — Windows (x64 / ARM64), macOS (Apple Silicon / Intel), Linux (AppImage, deb, rpm).

## What's new

### Desktop app

- **Agents panel** with analytics, detail view, tester, and **custom sub-agent** registration form.
- **Scripts tab** consolidation for script-related workflows.
- **Render queue panel** in the viewport (`queue_render` / `cancel_render`).
- **Scene presets** section in the Scene panel (save / load / delete via SQLite).
- Shared **PanelShell** component and updated learning dashboard styling.

### Daz Studio bridge ([plugin-v0.5.5](https://github.com/millsydotdev/DAZPilot/releases/tag/plugin-v0.5.5))

- Bridge pane: status, log, **Refresh**, **Send viewport to DazPilot**, and **Copy diagnostics**.
- **`bridge_commands.manifest`** shipped with the plugin release for schema parity.
- Viewport context can be pushed to the running DazPilot app for analysis.

### Docs & platform

- **[Platform matrix](https://github.com/millsydotdev/DAZPilot/blob/master/docs/PLATFORM_MATRIX.md)** — supported OS/arch combinations for app and plugin.

## Improvements

- UI layout consolidation (removed duplicate header / preset / tutorial panels in favor of integrated sections).
- Release CI now uploads **only installable artifacts** (no empty `target/` or `resources/` folders).
- macOS auto-update archives are **named per architecture** when both Intel and Apple Silicon builds share the same base filename.
- Windows plugin release is **x64-only** (DAZ SDK ships x64 libraries; Daz Studio on ARM64 Windows uses the same x64 DLL).

## Fixes

- Bridge plugin **builds cleanly** on MSVC (Qt 4–compatible pane; removed incomplete ImGui integration).
- CI correctly finds `DazPilotBridge.dll` under `dist/` / `dist/Release/`.
- GitHub release publish no longer fails on directory uploads or duplicate `DAZPilot.app.tar.gz` asset names.

## Install

### DazPilot desktop

Download the asset for your platform from this release:

| Platform | Typical assets |
| --- | --- |
| **Windows x64** | `.msi` or `_x64-setup.exe` |
| **Windows ARM64** | `_arm64-setup.exe` or `_arm64_en-US.msi` |
| **macOS Apple Silicon** | `_aarch64.dmg` or `DAZPilot_aarch64-apple-darwin.app.tar.gz` |
| **macOS Intel** | `_x64.dmg` or `DAZPilot_x86_64-apple-darwin.app.tar.gz` |
| **Linux x64** | `.AppImage`, `.deb`, or `.rpm` (`amd64`) |
| **Linux ARM64** | `.AppImage`, `.deb`, or `.rpm` (`aarch64`) |

In-app updates use **`latest.json`** on this release page.

### Daz Studio bridge plugin

Use the separate **[plugin-v0.5.5](https://github.com/millsydotdev/DAZPilot/releases/tag/plugin-v0.5.5)** release:

1. Download **`DazPilotBridge-x64.dll`** (or `DazPilotBridge.dll` — same x64 build).
2. Copy into your Daz Studio **plugins** folder.
3. Restart Daz Studio.
4. Open the **DazPilot Bridge** pane from the DazPilot menu.

The bridge listens on **`127.0.0.1:8765`**; the app connects as a client.

## Requirements

- **DAZ Studio 4.5+** (with bridge plugin installed for live scene control).
- **Windows / macOS / Linux** — see [platform matrix](https://github.com/millsydotdev/DAZPilot/blob/master/docs/PLATFORM_MATRIX.md) for arch support.
- Optional: local GGUF model or Ollama for AI features (see [README](https://github.com/millsydotdev/DAZPilot#readme)).

## Known limitations

- Bridge plugin is **Windows x64 only** at this time; macOS/Linux bridge ports are not in this release.
- Live validation against a full Daz Studio session remains the main manual acceptance step ([ACCEPTANCE.md](https://github.com/millsydotdev/DAZPilot/blob/master/docs/ACCEPTANCE.md)).
